## 2. データ管理

| No. | 項目 | 回答 |
|-----|------|------|
| 2.1 | データの保存場所 | **お客様環境内のみ** — PostgreSQL データベース |
| 2.2 | クラウドへのデータ送信 | **既定は無し**（完全オフライン動作可能。OpenAI / Anthropic / Gemini 等のクラウドLLMをオプションで利用する場合のみ、明示的な設定で外部APIに送信） |
| 2.3 | テレメトリ / 利用統計の送信 | **なし** |
| 2.4 | ライセンス認証の通信 | **なし** — Hardware UUIDベースのオフライン認証 |
| 2.5 | データの暗号化（保存時） | PostgreSQLの暗号化機能に依存 |
| 2.6 | データの暗号化（通信時） | Nginxリバースプロキシによる HTTPS 対応可能 |
| 2.7 | データバックアップ | `pg_dump` による標準的なPostgreSQLバックアップ |

## 3. 認証・アクセス制御

| No. | 項目 | 回答 |
|-----|------|------|
| 3.1 | 認証方式 | **ローカル（ID/パスワード）/ LDAP / OIDC から選択** |
| 3.2 | LDAP / Active Directory 対応 | **対応** — AD / OpenLDAP（python-ldap3）。初回ログイン時にユーザー自動作成 |
| 3.3 | OIDC / Azure AD 対応 | **対応** — Microsoft Entra ID。初回サインイン時にユーザー自動作成 |
| 3.4 | パスワードハッシュ | **bcrypt（passlib, 12ラウンド）** |
| 3.5 | ロールベースアクセス制御 | **あり** — ADMIN / SUPER / USER の3段階 |
| 3.6 | セッション管理 | **JWT (HS256) + HTTP-only Cookie**（有効期間365日） |
| 3.7 | 管理者アカウントの保護 | 最後のADMINは削除・降格不可 |
| 3.8 | デフォルトパスワード | 初期: admin@local / admin123 — **初回ログイン後に変更必須** |
| 3.9 | ユーザー数制限 | ライセンスに基づく上限あり |
| 3.10 | AD/OIDC環境の管理者アクセス | admin@local は常にローカル認証可能 |
| 3.11 | 共有ディレクトリACL / runAs | Bot・パイプラインに「タグ単位の共有 + runAs（実行ユーザーのなりすまし）」を設定可能 |

## 4. ネットワーク

| No. | 項目 | 回答 |
|-----|------|------|
| 4.1 | 必要なポート | **DigitalBase (API + Web): 8000**, PostgreSQL: 5432, Ollama: 11434（vLLM版: 8080, 8081） |
| 4.2 | 外部への通信 | **不要**（初回モデルダウンロード時のみインターネット必要。クラウドLLM利用時を除く） |
| 4.3 | HTTPS対応 | Nginxリバースプロキシにて対応可能 |
| 4.4 | ファイアウォール設定 | ufw によるIP制限・ポート制御をサポート |
| 4.5 | バインド設定 | **デフォルトは `0.0.0.0`（LAN内の他端末からアクセス可能）**。`API_HOST=127.0.0.1` に変更すると同一サーバー内のみに制限可 |

> ⚠️ デフォルト設定では LAN 内の他端末からアクセスできます。同一サーバー内に閉じたい場合は `~/.local/db/.env` の `API_HOST` を `127.0.0.1` に変更してください。
