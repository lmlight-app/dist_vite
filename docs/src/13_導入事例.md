## 事例1: 製造業 — 社内ナレッジ検索の効率化

### 企業プロフィール

| 項目 | 内容 |
|------|------|
| 業種 | 製造業 |
| 従業員数 | ＿＿名 |
| 導入規模 | ＿＿ライセンス |
| 利用エディション | Ollama版 |

### 課題

- 社内マニュアルが数百ファイルに分散し、必要な情報を見つけるのに時間がかかる
- ベテラン社員のノウハウが属人化している
- 外部クラウドAIにはセキュリティ上、社内データをアップロードできない

### 導入した機能

| 機能 | 用途 |
|------|------|
| RAG | 社内マニュアル・技術資料のAI検索 |
| AIチャット | 業務に関する質問応答 |
| 文字起こし | 会議録音のテキスト化 |

### 導入効果

| 指標 | Before | After |
|------|--------|-------|
| 資料検索時間 | ＿＿分/回 | ＿＿分/回 |
| 問い合わせ対応 | ＿＿件/月 | ＿＿件/月 |

## 事例2: IT企業 — Active Directory連携による全社展開

### 企業プロフィール

| 項目 | 内容 |
|------|------|
| 業種 | 情報通信業 |
| 従業員数 | ＿＿名 |
| 導入規模 | ＿＿ライセンス |
| 利用エディション | vLLM版 |

### 課題

- 部門ごとにAIツールがバラバラで管理が困難
- 社員のアカウント管理を既存ADに統一したい
- 高速な推論が必要（同時利用ユーザーが多い）

### 導入した機能

| 機能 | 用途 |
|------|------|
| AIチャット | 全社員向けAIアシスタント |
| RAG | 部門別のナレッジベース（タグで共有範囲制御） |
| LDAP認証 | Active Directoryとのシングルサインオン |
| SQLエージェント | 社内DBへの自然言語クエリ |

### 導入効果

| 指標 | Before | After |
|------|--------|-------|
| AI利用率 | ＿＿% | ＿＿% |
| アカウント管理工数 | ＿＿h/月 | ＿＿h/月 |

## 事例3: サービス業 — ETLパイプラインによる業務データ自動処理

### 企業プロフィール

| 項目 | 内容 |
|------|------|
| 業種 | サービス業 |
| 従業員数 | ＿＿名 |
| 導入規模 | ＿＿ライセンス |
| 利用エディション | vLLM版 |

### 課題

- 複数システムからのデータ集約に手作業が多く、毎月数日を要していた
- CSVやExcelで届くデータのフォーマットが統一されておらず、手動変換が必要
- AIを使ったデータ分類・要約を自動化したいが、外部クラウドにデータを送れない

### 導入した機能

| 機能 | 用途 |
|------|------|
| ETLパイプライン | CSV/Excel取込 → AI変換（map/filter/reduce） → DB格納の自動化 |
| AIチャット | パイプライン結果の確認・再分析 |
| 承認フロー | パイプライン実行結果の上長承認 |
| クラウドLLM連携 | 高精度が必要な分類タスクでGPT-4oを併用 |

### 導入効果

| 指標 | Before | After |
|------|--------|-------|
| 月次データ集約工数 | ＿＿h/月 | ＿＿h/月 |
| データ変換エラー率 | ＿＿% | ＿＿% |
