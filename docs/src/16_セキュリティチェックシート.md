## 4. 認証・アクセス制御

| No. | 項目 | 回答 |
|-----|------|------|
| 4.1 | 認証方式 | **ローカル（ID/パスワード）/ LDAP / OIDC から選択** |
| 4.2 | LDAP / Active Directory 対応 | **対応** — AD / OpenLDAP。初回ログイン時にユーザー自動作成 |
| 4.3 | OIDC / Azure AD 対応 | **対応** — Microsoft Entra ID。初回サインイン時にユーザー自動作成 |
| 4.4 | パスワードハッシュ | **bcrypt（12ラウンド）** |
| 4.5 | パスワードポリシー | お客様のLDAP/AD側のポリシーに準拠（ローカル認証時はアプリ側で管理） |
| 4.6 | ロールベースアクセス制御 | **あり** — ADMIN / SUPER / USER の3段階 |
| 4.7 | セッション管理 | **JWT + HTTP-only Cookie**（有効期間365日） |
| 4.8 | 管理者アカウントの保護 | 最後のADMINは削除・降格不可 |
| 4.9 | デフォルトパスワード | 初期: admin@local / admin123 — **初回ログイン後に変更必須** |
| 4.10 | ユーザー数制限 | ライセンスに基づく上限あり |
| 4.11 | AD/OIDC環境の管理者アクセス | admin@local は常にローカル認証可能 |
| 4.12 | 同時ログイン制御 | JWT方式のため複数端末からの同時ログイン可能 |
| 4.13 | アカウントロックアウト | LDAP/AD側のポリシーに準拠（ローカル認証時は現在未実装） |

## 6. アプリケーションセキュリティ

| No. | 項目 | 回答 |
|-----|------|------|
| 6.1 | SQLインジェクション対策 | **Prisma（フロントエンド）+ SQLAlchemy（API）によるパラメータ化クエリ** |

| 6.5 | ファイルアップロード制限 | 最大100MB（音声ファイル） |
| 6.6 | API ドキュメント保護 | Swagger UI / ReDoc は Basic認証で保護 |
| 6.7 | 入力値バリデーション | FastAPI の Pydantic モデルによる型検証 + フロントエンド側バリデーション |
| 6.8 | エラーハンドリング | 内部エラーの詳細はクライアントに返さない（500エラー時はジェネリックメッセージ） |
| 6.9 | 依存パッケージの脆弱性管理 | npm audit / pip audit による定期チェック |
